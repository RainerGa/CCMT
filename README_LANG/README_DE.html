Contact Center Minitools (CCMT)


CCMT bietet Ihnen kleine Tools, um Ihnen die tägliche Arbeit zu erleichtern. Sei es in einem Call Center/Contact Center, in einer Hotline oder in ihrem normalen Büroalltag.


Tool Number 1: CCMT Phone Number to Ticket

Ziel dieses Werkzeugs ist es, einen ankommenden Anruf einem Kunden zuzuordnen und das dazugehörige Ticket direkt in der dazugehörigen Anwendung zu öffnen.


Dafür benötigen Sie natürlich einen Telefon Software Client an dem die Telefonnummer des Kunden angezeigt wird.


Besagte Anwendung wird mittels Link geöffnet und die gefundene Ticketnummer wird diesem Link mitgegeben.


Statt einer Ticketnummer können auch andere Daten hinterlegt werden!

Statt der Verlinkung zu einem Ticketsystem können sie beispielsweise auch eine EXE Datei starten und ausführen.


Die jeweils anzupassenden Skriptpassagen sind leicht zu finden und gut kommentiert.


Abstraktere Programmfunktion

Bildschirmbereich wird analysiert (OCR)

Daten werden genutzt um in einer Textdatei ein Mapping durchzuführen

Die mittels Mapping gefundenen Daten werden genutzt um

Mittels Link eine andere Anwendung zu öffnen


Grundsätzlich sind damit vielerlei Anwendungsfälle möglich


FAQ

Warum wurde das Programm in Powershell geschrieben?

Grund dafür ist, das in vielen Firmen die Installation von Anwendungen verboten ist. Das starten eines Skriptes aber erlaubt ist.


Warum wird nicht direkt die API des Telefon Software Clients verwendet?

Hersteller haben häufig Restriktionen bei der Verwendung der API/Schnittstellen. Es gibt Hersteller, bei denen die API Verwendung explizit lizensiert werden muss. Dazu währe die Anwendung dann auf einen einzigen Hersteller begrenzt. Ziel ist es aber, eine Anwendung bereitzustellen, welche immer funktioniert.


Warum ist diese Anwendung entstanden, dafür gibt es doch RPA (Robott Process Automation) Software

RPA Software ist häufig komplex UND eine Installation ist bei vielen Arbeitgebern nicht so einfach möglich. Die Herausforderung ist es also, eine Anwendung bereitzustellen, die einfach in jeder Umgebung genutzt werden kann/darf, keine Kosten verursacht UND ich die Möglichkeit habe einmal etwas in Powershell zu programmieren (meine eigentliche Programmiersprache ist Java).
