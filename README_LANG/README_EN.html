Contact Center Minitools (CCMT)


CCMT offers you small tools to make your daily work easier. Be it in a call center / contact center, in a hotline or in your normal everyday office life.


Tool Number 1: CCMT Phone Number to Ticket

The aim of this tool is to assign an incoming call to a customer and to open the associated ticket directly in the associated application.


To do this, of course, you need a telephone software client on which the customer's telephone number is displayed.


Said application is opened via the link and the ticket number found is given to this link.


Instead of a ticket number, other data can also be stored!

Instead of linking to a ticket system, you can also start and run an EXE file, for example.


The script passages to be adapted are easy to find and well commented on.


Abstraktere Programmfunktion

Screen area is analyzed (OCR)

Data is used to carry out a mapping in a text file

The data found by means of mapping are used to

Open another application using the link


In principle, many applications are possible with this


FAQ

Why was the program written in Powershell?

The reason for this is that the installation of applications is prohibited in many companies. But starting a script is allowed.


Why is the API of the telephone software client not used directly?

Manufacturers often have restrictions when using the API / interfaces. There are manufacturers for whom the use of the API must be explicitly licensed. For this purpose, the application would then be limited to a single manufacturer. However, the aim is to provide an application that always works.


Why did this application come about? RPA (Robott Process Automation) software is available for that

RPA software is often complex AND installation is not that easy for many employers. So the challenge is to provide an application that can / may be used in any environment, incurs no costs AND I have the opportunity to program something in Powershell (my actual programming language is Java).
